# Asset Generation & Sourcing — Bachelor Imperium IV

What art/audio the dashboard needs, beyond the 30 cards already generated.

## 1. Card images — DONE

All 30 cards live in `../card_prompts/generated/`. The build/setup step copies
them into `./cards/`. No new AI generation needed for cards.

If you want to regenerate any card, see `../card_prompts/*.md` for the prompts.

## 2. Wheel graphic — NO AI NEEDED

Drawn in pure CSS/SVG by the projector code:
- 3 equal-angle slices (will look unequal because of weighting? No — slices are
  visually 120° each regardless of probability. The weights are silent.)
- Colors: red (#8B1A1A), gold (#B8860B), purple (#5B21B6).
- Labels: "PUBLIC" / "SECRET" / "TARGETED" centered on each slice.
- Pointer at top, fixed.
- Spin animation: CSS `@keyframes` with cubic-bezier deceleration.

## 3. Background photo album — YOU DROP IN

Place any number of `.jpg` / `.png` files in `./photos/`. The projector
slideshow auto-discovers them on load (via a manifest file `photos/index.json`,
generated by a tiny `npm run build-photos` script — or written by hand).

Suggested content:
- Old photos of Jeroen (childhood, school, group photos)
- Photos of Marijke (sparingly — she's a forbidden word, but the photo is fine)
- Group photos of the friend circle
- Photos from previous TI4 sessions if any
- Any cottage / Ardennes scene-setters

Aspect ratio: doesn't matter, the slideshow `object-fit: cover`s them.

**Privacy**: photos sit in the public GitHub repo. If you want them private,
drop them in a `photos-private/` subfolder that's `.gitignore`d, and serve them
from a local file path on the casting device (Z Fold storage).

## 4. Sound effects — SOURCE FROM FREESOUND.ORG

Three small MP3s in `./sounds/`. Recommended sources (all CC0 or CC-BY):

| File | Description | Search query on freesound.org |
|---|---|---|
| `bell.mp3` | Single soft council/temple bell ring. ~2s. Plays when wheel lands. | `temple bell single` or `meditation bell` |
| `drumroll.mp3` | Short ~2s drumroll, lands with a cymbal hit. Plays under wheel spin. | `short drumroll cymbal` |
| `applause.mp3` | Quick polite applause clip, ~3s, no whoops. For the door-applause flash. | `polite applause short` |

Keep file sizes < 200 KB each. Convert to MP3 if needed (audacity, online).
Volume normalized to ~80%; dashboard plays them via `<audio>` with a master
volume slider in operator config.

## 5. Final-reveal flair — OPTIONAL

For Sunday's winner crowning:
- A 🏆 emoji is enough; no graphic needed.
- Confetti: pure CSS particle animation, no asset.
- Optional music sting: search "fanfare short" on freesound.org and drop in
  `sounds/fanfare.mp3`. Otherwise silent.

## 6. Favicon / page title — OPTIONAL POLISH

A 32×32 favicon helps you find the operator tab among many. Quick option:
single emoji as favicon via `data:` URL in `<head>`:

```html
<link rel="icon" href="data:image/svg+xml,<svg xmlns=%22http://www.w3.org/2000/svg%22 viewBox=%220 0 100 100%22><text y=%22.9em%22 font-size=%2290%22>🛸</text></svg>">
```

Page titles: `🛸 Bachelor Imperium IV` (projector), `🎛 Operator Console` (op).

## 7. AI image generation — NOT NEEDED for the dashboard itself

Everything visual in the dashboard is either:
- A generated card (already done)
- A real photo (you supply)
- CSS/SVG (drawn in code)

No additional AI prompts to feed to an image model for the dashboard build.
